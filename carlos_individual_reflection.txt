Individual Reflection

For this project I worked primarily on improving robustness and observability across several server components, including rpc_protocol.py, rpc_server.py, sensor_server.py, pubsub.py, and notifier.py. My focus was ensuring the system behaved reliably under load and that failures or bottlenecks could be detected during testing.

One change I implemented was optimizing the _recv_exactly() method in rpc_protocol.py. The original implementation repeatedly concatenated byte strings when receiving framed RPC messages, which creates unnecessary allocations under heavy request rates. I replaced this approach with a bytearray buffer, preserving the same protocol behavior while reducing overhead during high-throughput RPC load tests.

I also strengthened request validation in rpc_server.py by adding explicit type checks and structured error responses when malformed RPC frames are received. This prevents invalid requests from propagating deeper into the server logic and ensures that clients receive clear error messages instead of unexpected failures.

A bug I fixed involved the pub/sub notifier delivery loop. Previously the loop caught all exceptions, which could hide real runtime errors during event delivery. I corrected this by catching only queue.Empty timeouts, allowing unexpected exceptions to surface while still handling normal subscriber delays.

Finally, I added lightweight metrics to improve observability of the system under load. In sensor_server.py, I added counters for dropped updates when the bounded sensor queue becomes full. In pubsub.py, I implemented per-subscriber and global counters for dropped events when back-pressure occurs. These metrics allowed us to observe how the system behaved during the sensor stress tests without changing the core messaging protocol.
